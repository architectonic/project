# Standalone Project

## Spawn only a project when

- one initiative needs continuity across sessions or agents;
- source files, decisions, risks, and unknowns need a stable map;
- the project can inherit authority from the surrounding repository or human operator;
- no broader organization constitution or actor registry is needed.

## Spawn a project system when

- the project owns a substantial knowledge corpus;
- recurring procedures need verification and failure handling;
- drift, failures, and obsolete artifacts need deliberate upkeep.

```text
project-system = project + knowledge + skills + meta
```

## Add organization layers when

- several projects share governance or actors;
- decision, delegation, spending, privacy, or stopping authority is ambiguous;
- organization-level rules must survive project retirement.

## Do not spawn a project when

- the work is disposable and finishes in one bounded session;
- an existing project already owns the context;
- the only need is one reusable skill or one stable corpus.
